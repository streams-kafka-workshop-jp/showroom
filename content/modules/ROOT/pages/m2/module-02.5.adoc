== 実際のKafkaアプリケーション: Eコマース活動トラッキング

このセクションでは、リアルタイムデータストリーミングと処理のユースケースを示すためにeコマースアプリケーションを探索します。このワークショップで使用されるGlobex webアプリケーションはデモアプリであり、実際の小売アプリケーションであるとは主張しませんが、実際の小売webアプリのように感じさせるのに十分な機能を備えています。

* ブラウザから、link:{openshift_cluster_console}/topology/ns/*globex-{user_name}*?view=graph[OpenShift Cluster Console, window="_console"]に移動します。ユーザー名とパスワード(`{user_name}`/`{password}`)でログインします。

Globex web-appデプロイメントは以下で構成されています:
+

[frame=all, grid=all]
|===
|*Service* | *説明*

| globex-web
| *Globex webアプリケーション* のwebフロントエンド。Node.jsとAngular UIで実行されています。

| globex-store-app
| *Globex retailアプリ* のバックエンドアプリケーション。Quarkusで書かれており、catalog、inventory、cart、orderサービスを提供します。

| *globex-store-appアプリケーション* が使用するPostgreSQLデータベース。catalog、inventory、customer、order情報を保存します。

| activity-tracking
| このmicroserviceには、UIから生成されるユーザーアクティビティイベント用のRESTエンドポイントがあります。これらのイベントはKafka topicに送信されます。

| recommendation-engine (このサービスは後のモジュールで詳しく探索します。)
| イベントstreamのリアルタイム処理を行うアプリケーションの例* このアプリケーションはKafka Streams(イベントストリーミングlibrary)を使用して、activity trackingサービスによって生成されたイベントのstreamを処理し、webサイトで最も人気のある製品のリストを構築します* このリストはRESTエンドポイントとして公開され、フロントエンドによって呼び出されて注目の製品を表示します。

|===

== ユーザーの_いいね_アクティビティを生成する


* Globex webサイトで、トップメニューの *Cool Stuff Store* リンクをクリックします。ここから、製品catalogを閲覧したり、製品詳細ページを開いたり、ショッピングカートにアイテムを追加したりできます。
+
image::m2/globex-view-catalog.png[]
+
NOTE: 製品catalogの読み込みに初回は数秒かかる場合があります。

* ユーザーアクティビティイベントを生成するには、製品の横にあるimage:m2/globex-product-like.png[]シンボルをクリックします。これを3〜4つの製品に対して繰り返すことができます。

* すべての_いいね_アクションは、_activity tracking_サービスによってキャプチャされ、Kafka topicに送信されるユーザーアクティビティイベントを生成します。

* Kafka brokerは *kafka-{user_name}* namespaceにインストールされています。同じnamespaceに、Kafka topicを表示してconsumer groupを閲覧するためのweb UIであるstreams for Apache Kafka consoleもインストールされています。

* クリックしてlink:https://amq-streams-console-{user_name}.{openshift_cluster_subdomain}[streams for Apache Kafka console, window="_amqstreams"]に移動します。これにより、streams for Apache Kafka consoleログインページにリダイレクトされます。このワークショップの目的では、 *Click to login anonymously* を選択してconsoleにアクセスします

* 左側の *Topics* メニューを選択して、topicのリストを表示します。
+
image::m2/globex-tracking-topic-console.png[]

* それらのtopicの1つは *globex.tracking* と呼ばれ、ユーザーアクティビティイベントを含むtopicです。topic名をクリックして、topicの詳細を確認します。webサイトでいくつかの製品を「いいね」した場合、topicにはいくつかのメッセージが含まれているはずです。
+
image::m2/amqstreams-globex-tracking.png[]

* リストされているメッセージのいずれかをクリックして、その内容を確認します。この場合、各メッセージの本文は、Globex webアプリケーションからのユーザーアクティビティイベントのJSON構造で構成されています。
+
image::m2/amqstreams-expand-message.png[]

* activity trackingサービスは、これらのイベントのproducerです。いいねなどのすべてのイベントは、Kafka Producer APIを使用してリアルタイムで *globex.tracking* topicに送信されます。


これは、Kafka topicにイベントを送信するKafka producerアプリケーションのシンプルな実世界の実用例です。activity trackingとrecommendation engineサービス(stream processingモジュールで探索します)の両方は、cloud-nativeアプリケーションを構築するためのJavaフレームワークであるQuarkusを使用して実装されており、Kafka Producer APIを直接使用する場合と比較して、Kafkaとの対話を簡素化します。
