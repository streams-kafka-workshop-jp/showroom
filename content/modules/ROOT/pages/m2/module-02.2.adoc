== シンプルなProducerアプリケーションの実行

前のセクションで分析したシンプルなストリーミングアプリケーションを実行してみましょう。

* Devspaces IDEで、左上隅のメニューをクリックして新しいターミナルを開きます。次に*Terminal > New Terminal.* を選択して新しいターミナルウィンドウを起動します。
+
image::m2/devspaces-terminal.png[]

* ターミナルウィンドウで、次のコマンドを実行してプロジェクトディレクトリに移動します。
+
[source,bash,role="execute",subs=attributes+]
----
cd module-kafka-api/
----

NOTE: ブラウザがターミナルへの貼り付けの許可を求めた場合は、*allow* をクリックしてください。

* 次のコマンドを使用して*kafka-{user_name}* projectにログインします。
+
[source,bash,role="execute",subs=attributes+]
----
oc project kafka-{user_name}
---- 

// * The terminal has multiple java versions installed. For the sake of consistency across the workshop make sure you are using java 17 by running the following command.
// +
// [source,sh,role="execute",subs=attributes+]
// ----
// sdk use java 17.0.3-tem
// java --version
// ----

*  ワークショップ全体の一貫性のために、次のコマンドを実行してjava 21を使用していることを確認してください。
+
[source,sh,role="execute",subs=attributes+]
----
sdk install java 21.0.2-tem && sdk use java 21.0.2-tem
----

* プロンプトが表示されたら、pasteボタンをクリックします。


* ターミナル出力でjava 21を使用していることを確認してください。

* 左側のfile explorerで、`module-kafka-api`フォルダを展開し、次に`src/main/resources`フォルダを展開します。`application.properties`ファイルをクリックして開きます。

* bootstrap serversのプレースホルダ値(6行目)を次の値に置き換えます。
+
[source,bash,role="execute",subs=attributes+]
----
kafka-kafka-bootstrap.kafka-{user_name}.svc.cluster.local:9092
----

* bootstrap serversの値に対応するユーザー名が含まれていることを再度確認してください
+
image::m2/app-properties-bootstrap.png[]




* それでは、producerアプリケーションを実行してみましょう。次のコマンドを実行します。これによりアプリケーションのビルドが開始され、完了するとインタラクティブアプリケーションが実行されます。
+
[source,bash,role="execute",subs=attributes+]
----
./run.sh
----

* 次のようなプロンプトが表示されるはずです。
+
[source,bash]
----
========================================
   Kafka Java Tutorial Application
========================================

Main Menu:
1. Run Producer
2. Run Consumer
3. Exit

Enter your choice:
----

* `1`を入力して*オプション* 1を選択し、Enterキーを押します。

* *Enter* キーを押し続けてデフォルト値を受け入れます。これによりproducerがメッセージを送信するトリガーになります。

* ターミナル出力にエラーが継続的にスローされることを確認できます。エラーを確認した後、*ctrl+c* を押してproducerを停止します。

* エラーメッセージは次のようになり、topic `tutorial-topic`が存在しないことを示しています。
+
[source,bash]
----
Error while fetching metadata with correlation id 57 : {tutorial-topic=UNKNOWN_TOPIC_OR_PARTITION}
[kafka-producer-network-thread | producer-1] WARN org.apache.kafka.clients.NetworkClient - [Producer clientId=producer-1] Error while fetching metadata with correlation id 58 : {tutorial-topic=UNKNOWN_TOPIC_OR_PARTITION}
----

* *ctrl+c* を押してproducerを停止します。

* このclusterでは*auto.create.topics.enable* プロパティが`false`に設定されています。これは、producerまたはconsumerが存在しないtopicにアクセスしようとした時に、*新しいtopicが自動的に作成されるかどうかを制御する* Kafka brokerの設定です。

* この設定は、producerまたはconsumerコードのタイプミスからジャンクtopicが作成されることを防ぐため、有用な設定です。

* devspacesターミナルで`clear`と入力してEnterキーを押し、ターミナル出力をクリアします。

* それでは、topicを作成しましょう。次のコマンドを実行します。
+
[source,bash,role="execute",subs=attributes+]
----
oc run -n kafka-{user_name} topic-create --rm --restart=Never --quiet --attach \
--image=registry.redhat.io/amq-streams/kafka-40-rhel9:3.0.0 -- \
bash -lc '/opt/kafka/bin/kafka-topics.sh --bootstrap-server kafka-kafka-bootstrap:9092 --create --topic tutorial-topic --partitions 3 --replication-factor 1'
----

* topicが作成されたら、インタラクティブアプリケーションを再度実行します。
+
[source,bash,role="execute",subs=attributes+]
----
./run.sh
----

* `1`を入力してEnterキーを押し、producerを実行します。

* producerは、keyとvalueを持つ*10個のメッセージ* を自動的に送信するように設定されています。各メッセージは約*30バイト* です。

* 最初に*バッチサイズ* を入力するように求められます。アプリケーションのデフォルト値は*16384バイト* に設定されています。Enterキーを押してデフォルト値を受け入れます。



* 次のオプションは*linger.ms* です。アプリケーションのデフォルト値は0で、これはproducerがメッセージを送信する前にメッセージが蓄積されるのを待たないことを意味します。`30000`を入力してEnterキーを押します。以下の値を入力してEnterキーを押します。
+
[source,bash,role="execute",subs=attributes+]
----
30000
----

* これにより、producerはバッチが満杯になるか30秒経過するまで待機してから送信するようになります。

* メッセージがすぐには送信されないことがわかります。30秒後にのみメッセージがtopicに送信されます。

* 30秒待つと、30秒が経過した後にメッセージが送信されることがわかります。
+
[source,bash]
----

* 送信されるメッセージと比較してバッチサイズが非常に大きいため、producerはバッチを送信する前にlinger.msが経過するのを待ちます。

* 次に、*Enter* キーを押してメインメニューに戻ります

* `1`を押して再度*Enter* キーを押し、producerを再度実行します。

* バッチサイズ(16 KB)とlinger.ms(0)秒の*デフォルト値* を受け入れます。Enterキーを押す前に、何が起こるかを推測し、出力を観察してください。

* それではEnterキーを押してください。メッセージが*待ち時間なしで即座に送信される* ことを確認できます。これは、linger.msが0秒に設定されているためです。producerはメッセージを送信する前にメッセージが蓄積されるのを待ちません。

* Enterキーを押して

* `3`を入力して再度Enterキーを押し、アプリケーションを終了します。
