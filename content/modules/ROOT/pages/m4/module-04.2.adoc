== Change Eventがキャプチャされたことを確認

* connectorが動作していることを確認する1つの方法は、Globex retailアプリケーションを通じていくつかの注文を作成することで、データベースにchange eventを作成することです。

* Globex retailアプリケーションを指しているブラウザタブをクリックします。アプリケーションでブラウザタブを開いていない場合は、link:{globex_web_url}[Globex retail application, window="_globex"]に移動してアプリケーションを起動します。

* アプリケーションの右上隅にあるログインボタンをクリックします。
+
image::m4/globex-login.png[]

* ここにリストされているユーザー名のいずれかでログインします `asilva`/`cjones`/`asanders`/`pwong`/`mmiller`。パスワードは `openshift` です。
+
image::m4/login_credentials_page.png[]

* ログインしたら、トップバーの *Cool Stuff Store* リンクをクリックしてストアに移動します。
+
image::m4/cool-stuff-store.png[]

* 選択したアイテムをクリックして製品詳細ページを開き、 *Add to Cart* ボタンをクリックしてアイテムをショッピングカートに追加します。より多くの注文をシミュレートできるように、複数のアイテムをカートに追加してください。
+
image::m4/add_to_cart.png[]

* 右上隅のショッピングカートアイコンをクリックしてショッピングカートを開きます。
+
image::m4/cart_icon.png[]

* カート内のアイテムの数量を自由に調整してください。 *Proceed to Checkout* ボタンをクリックしてチェックアウトに進みます。
+
image::m4/proceed_to_checkout.png[]

* *Autofill Form* ボタンをクリックしてチェックアウトフォームを自動入力します
+
image::m4/autofill.png[]

* *Submit Order* をクリックして注文を送信します。
+
image::m4/submit_order.png[]

* 次のような注文確認ページが表示されます。
+
image::m4/order_confirmation_page.png[]

* Globex retailアプリケーションでいくつかの注文を作成したので、Debezium connectorがデータベース内の対応するchange eventをキャプチャし、適切なKafkaトピックに生成したことを確認しましょう。

* streams for Apache Kafka console UIを指しているブラウザタブを開きます。タブを閉じた場合は、 https://amq-streams-console-{user_name}.{openshift_cluster_subdomain}[streams for Apache Kafka console, window="_amqstreams"] に移動します。

* Topicsページから、*globex.updates.public.orders* トピックを開き、トピックに1つのメッセージが含まれていることを確認します。
+
image::m4/amq-console-debezium-topic-orders.png[]

* メッセージをクリックして展開します。顧客のものと非常に似たchange event構造が表示されるはずです。ただし、操作が _create_ の場合は '*c*' であることに注意してください。これは、change eventが *order* テーブルの新しいレコードに対応するため、期待通りです。
+
image::m4/amq-console-debezium-topic-order-details.png[]

* streams for Apache Kafka Consoleのトピックページに戻り、今度は *globex.updates.public.line_item* トピックを開きます。以前に作成した注文のアイテムごとに1つのメッセージが表示されるはずです。
+
image::m4/amqconsole-debezium-topic-line-items.png[]


* より多くの注文をシミュレートしたい場合は、OpenShiftの _globex-{user_name}_ namespaceにデプロイされた _Order simulator_ アプリケーションを使用できます。

* ブラウザウィンドウで、OpenShiftコンソールを指しているタブを開きます。コンソールのタブが開いていない場合は、{openshift_cluster_console}/topology/ns/globex-{user_name}?view=graph[OpenShift console, window="_console"]に移動します。必要に応じて、ユーザー名とパスワード（`{user_name}`/`{password}`）でログインします。


// * Switch to the *globex-{user_name}* namespace by selecting the namespace from the namespace selection drop-down menu in the top left.
// +
// image::m4/openshift-console-developer-select-namespace.png[]


* Topologyビューで、以下に示す *order-simulator* デプロイメントの横にあるrouteリンクをクリックします。
+
image::m4/openshift-route-order-simulator.png[]


* これにより、シミュレーターのREST APIを示すSwagger UIページが開きます。+
*POST* リンクをクリックし、次に右側の *Try it out* リンクをクリックします。
+
image::m4/order-simulator-swagger-ui-post.png[]


* *Examples* ドロップダウンから、*random customers* を選択してランダムな顧客の注文を作成します。シミュレートする注文の数を自由に変更してください（デフォルトは50です）。
+
image::m4/order-simulator-random-customer.png[]

* *Execute* をクリックして、シミュレーターへのREST呼び出しを実行します。

* https://amq-streams-console-{user_name}.{openshift_cluster_subdomain}[streams for Apache Kafka console, window="_amqstreams"]で、*globex.updates.public.orders* と *globex.updates.public.line_item* トピックに新しいメッセージが生成されていることを確認します。

* これは、Debeziumをリレーショナルデータベースからchange eventをキャプチャし、それらをリアルタイムでKafkaトピックに生成するために使用できる方法を示しており、イベント駆動型アーキテクチャとデータ統合シナリオを可能にします。
