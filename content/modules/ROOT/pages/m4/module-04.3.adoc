== Change Data Captureを使用したGlobex Loyalty Program

* Change Data Captureを使用するサンプルアプリケーションを理解するために、顧客の購入を追跡し、ロイヤルティポイントを付与するロイヤルティプログラムを作成します。ロイヤルティプログラムアプリケーションは、DebeziumトピックからのChange Eventを消費し、それに応じてロイヤルティポイントを更新します。

* 前のセクションから開いているlink:https://devspaces.{openshift_cluster_subdomain}[Devspaces IDE, window="devspaces"]Devspaces IDEのブラウザタブに移動します。

* *kafka-workshop/module-cdc* フォルダ内の *loyalty-deployment.yaml* ファイルを選択します

* bootstrap servers（28行目）のプレースホルダー値を次の値に置き換えます
+
[source,bash,role=execute,subs="attributes"]
----
kafka-kafka-bootstrap.kafka-{user_name}.svc.cluster.local:9092
----



* devspacesターミナルで次のコマンドを実行して、ロイヤルティプログラムアプリケーションをOpenShiftプロジェクトにデプロイします
+
[source,bash,role=execute,subs="attributes"]
----
oc apply -f module-cdc/loyalty-deployment.yaml -n globex-{user_name}
----

* これにより、DebeziumトピックからのChange Eventを消費し、それに応じてロイヤルティポイントを更新するシンプルなQuarkusアプリケーションがデプロイされます。デプロイメントステータスを確認し、アプリケーションが実行されていることを確認します。
+
[source,bash,role=execute,subs="attributes"]
----
oc get pods -n globex-{user_name} -l app=cdc-dashboard -w
----

* podのステータスが *Running* と表示されたら、devspacesターミナルで `ctrl+c` と入力します。

* アプリケーション用に作成されたrouteに移動することで、ロイヤルティプログラムダッシュボードにアクセスできます。

* link:https://cdc-dashboard-globex-{user_name}.{openshift_cluster_subdomain}[Loyalty Program Dashboard, window="loyalty"]を開いて、globex-{user_name}にデプロイされたロイヤルティプログラムダッシュボードにアクセスします

* このダッシュボードは、Globex retailアプリケーションデータベースからキャプチャされたChange Eventに基づいて、顧客のロイヤルティポイントのリアルタイムビューを構築します。*globex.updates.public.orders* と *globex.updates.public.line_item* トピックのメッセージを使用して、retailアプリケーションデータベースと直接やり取りすることなく、顧客の購入を追跡し、それに応じてロイヤルティポイントを更新します。
+
image::m4/loyalty-dashboard.png[]

* Globex retailアプリケーションでより多くの注文が作成されるほど、Debeziumトピックにより多くのChange Eventが生成され、ロイヤルティプログラムダッシュボードがリアルタイムでロイヤルティポイントを更新します。

* 前のセクションで示したように、link:https://order-simulator-globex-{user_name}.{openshift_cluster_subdomain}[Order Simulator, window="order-simulator"]を使用してGlobex retailアプリケーションでさらに多くの注文を生成し、ダッシュボードでロイヤルティポイントがリアルタイムで更新される様子を確認してください。


* ロイヤルティプログラムアプリケーションチームは、Globex retailアプリケーションデータベースと直接やり取りすることなく、Debeziumによって生成されたChange Eventを活用する独自のサービスとアプリケーションを構築できるようになりました。これにより、ロイヤルティプログラムがretailアプリケーションから切り離され、イベント駆動型アプリケーションの構築においてより柔軟性とスケーラビリティが実現されます。

* 別のユースケースとして、 *リアルタイム注文トラッカーの構築* があります。これは倉庫のスクリーンに表示され、作成された注文を倉庫チームが迅速に処理できるように着信注文を表示します。サンプルアプリケーションはlink:https://cdc-dashboard-globex-{user_name}.{openshift_cluster_subdomain}/warehouse[こちら, window="warehouse"]で確認できます。
+
image::m4/warehouse-dashboard.png[]

* ロイヤルティプログラムアプリケーションと注文トラッカーアプリケーションのソースコードは、link:https://github.com/streams-kafka-workshop/module-cdc[こちら, window="_github"]で自由に探索してください。

これで、DebeziumによるChange Data Captureに関するこのモジュールは終了です。このモジュールでは、リレーショナルデータベースからのChange Eventをキャプチャし、リアルタイムでKafkaトピックに生成するDebezium connectorをデプロイする方法を見てきました。また、これらのChange Eventを消費してイベント駆動型アプリケーションを構築するシンプルなアプリケーションを構築する方法も見ました。
