=== 基礎 - Kafka Clusterとは？

Kafka Clusterは、イベントを保存し移動する分散システム全体です。特定の役割を持つ複数のKafkaプロセスで構成されています。

Controllers (KRaft): Clusterのメタデータ（どのtopic/partitionが存在するか、どのbrokerがどのpartitionをリードするか、など）を保持し、leader選出を実行します。

Brokers: データを保存し、クライアントトラフィックを処理します。

=== Clusterが重要な理由：

* スケーラビリティ: brokerを追加することで、スループットとストレージが増加します。

* フォールトトレランス（耐障害性）: brokerをまたいでpartitionをReplicationすることで、ノード障害が発生してもデータや可用性が失われません。

* 一元的なbootstrap: クライアントは単一の”bootstrap”アドレス（例：my-kafka-kafka-bootstrap:9092）に接続し、残りのノードを発見します。

* OpenShift/AMQ StreamsでのKRaft + Node Pools:

** Kafka CRはCluster全体の設定（Kafkaバージョン、リスナー、設定）を定義します。

** KafkaNodePool CRsはcontrollerノードとbrokerノードのセットを作成します。

=== Brokerとは？

Brokerの役割：

* Topicのpartitionをディスク上に保存します（追記専用ログ、保持期間ポリシー付き）。

* produce（書き込み）とconsume（読み込み）リクエストを処理します。

* ピアとの間でpartitionをReplicationします。1つのbrokerがpartitionのleaderとなり、他のbrokerはfollowerとなります（replication factor > 1の場合）。

=== Cluster詳細 - ウォークスルー
このワークショップでは、3つのbrokerで構成されるKafka Clusterがすでに作成されています。このワークショップを通じてこのClusterを使用します。{streams} consoleを使用してClusterの高レベルな詳細を簡単に確認しましょう。

* https://amq-streams-console-{user_name}.{openshift_cluster_subdomain}/kafka/[こちら, window=”_amqstreams”]をクリックして{streams} consoleに移動します。

* consoleに匿名でログインします。
+
image::m1/console_anonymous.png[]

* *Clusters overview* タブをクリックします。
+
image::m1/console_clusters_overview.png[]

* *Kafka cluster details* セクションには、Cluster名、brokerの数、Kafkaバージョンなどの詳細が表示されます。
+
image::m1/cluster_details.png[]

* *Cluster connection details* タブをクリックして、Clusterへの接続方法の詳細を確認します。
+
image::m1/connectiond_details.png[]

* ここには、Clusterのすべてのbootstrap serverと、それらが使用する認証方法がリストされます。詳細については後ほど学習します。現時点では、これがClusterへの接続方法です。
