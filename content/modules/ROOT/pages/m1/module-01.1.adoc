== Kafka基礎 - Topics

TopicはKafkaの基本構成要素です。Topicは、グループ化したいメッセージの論理的なコレクションです。名前付きで追記専用のレコード（イベント/メッセージ）のストリームです。

image::m1/topic-image.png[]

=== なぜTopicが必要か？
TopicはProducer（書き込み側）とConsumer（読み込み側）を疎結合にします。多くのProducerが書き込み、多くのConsumerが読み込むことができます。それぞれ独立して、自分のペースで処理できます。

=== 主要な特性

. *不変、追記専用:* レコードはその場で更新されることはありません。

. *マルチサブスクライバー:* 任意の数のConsumerが同じTopicを読み込むことができ、互いに干渉しません。

. *保持期間ベース:* Kafkaは時間（例：7日間）やサイズ（例：100GB）に基づいてデータを保持します。「consumeされるまで」ではありません。


=== Topic内のレコード（メッセージ）

. *Key（オプション）:* partitioning とcompaction semanticsに使用されます。

. *Value:* ペイロード（JSON、Avro、Protobuf、bytes）。

. *Headers（オプション）:* メタデータ（trace ID、content-typeなど）。

. *Timestamp:* イベント時刻またはbroker時刻。


=== Topicの作成

* *上部のターミナル* で、以下のコマンドを使用してOpenShift Clusterにログインします。image:common/clipboard.png[width=30px]アイコンを使ってコピーしてください。
+
[source,sh,role=”execute”,subs=attributes+]
----
{login_command}
----

* `Use insecure connections? (y/n):`と表示されたら`y`と入力してEnterを押します。

* KafkaクライアントツールがインストールされたKafka toolboxコンテナを作成しましょう。 *上部のターミナル* で以下のコマンドを実行します。
+
[source,sh,role=”execute”,subs=attributes+]
----
oc run -n kafka-{user_name} kafka-toolbox \
  --image=registry.redhat.io/amq-streams/kafka-40-rhel9:3.0.0 \
  --restart=Never \
  --env=”BOOTSTRAP=kafka-kafka-bootstrap.kafka-{user_name}.svc:9092” \
  --env=”PATH=/opt/kafka/bin:/usr/local/sbin:/usr/local/bin:/usr/sbin:/usr/bin:/sbin:/bin” \
  -- sleep infinity
----

* toolboxコンテナが作成されるまで数秒待ちます。

* *上部のターミナル* で以下のコマンドを実行して、作成したtoolboxコンテナに接続します。
+
[source,sh,role=”execute”,subs=attributes+]
----
oc rsh -n kafka-{user_name} kafka-toolbox
----

* *上部のターミナル* で、以下のコマンドを使用して、3つのpartitionとreplication factorが1の`demo`という名前のKafka topicを作成します（replicationについては後ほど詳しく説明します）。topicが作成されたという確認メッセージが表示されるまで数秒待ちます。
+
[source,sh,role=”execute”,subs=attributes+]
----
kafka-topics.sh --bootstrap-server $BOOTSTRAP \
  --create --topic demo --partitions 3 --replication-factor 1
----

// NOTE: This command creates a temporary container with the Kafka client tools installed and executes the Kafka command to create the topic. The container is removed after the command is executed. This is method is an easy way to learn the basics without having to install the client tools on your local machine.


// * In the *upper terminal*, run the following command to see the the details of the created topic, including the number of partitions and replicas.
// +
// [source,sh,role=”execute”,subs=attributes+]
// ----
// oc run -n kafka-{user_name} topic-describe --rm -it --restart=Never \
//   --image=registry.redhat.io/amq-streams/kafka-40-rhel9:3.0.0 -- \
//   bash -lc '/opt/kafka/bin/kafka-topics.sh --bootstrap-server kafka-kafka-bootstrap:9092 --describe --topic demo'
// ----

* https://amq-streams-console-{user_name}.{openshift_cluster_subdomain}/kafka/[こちら, window=”_amqstreams”]をクリックして{streams} consoleに移動し、作成したtopicを確認します。このconsoleは、Kafka cluster、topic、メッセージなどを可視化および監視するためのWebベースのユーザーインターフェースです。

* consoleに匿名でログインします。
+
image::m1/console_anonymous.png[]

* 左側のメニューで *Topics* をクリックして、topicのリストを表示します。作成した`demo` topicが表示されるはずです。このtopicには3つのpartitionがあります。
+
image::m1/console_topics.png[]
