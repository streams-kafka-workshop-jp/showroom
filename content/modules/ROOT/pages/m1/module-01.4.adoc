== Kafka基礎 - Consumer（データが取り出される方法）
Kafka ConsumerはKafka topicからメッセージを読み取り、処理のためにアプリケーションコードに渡します。Consumerは以下を行うコンポーネントです。

* 1つ以上のtopicにサブスクライブする

* 割り当てられたpartitionからレコードを取得する

* 処理済みの内容を把握するためにoffset（位置）を追跡する

* 並列処理とフォールトトレランス（耐障害性）のためにconsumer groupを介して他のconsumerと調整する

* Consumerはtopic partition内の任意のoffsetからデータを読み取ることができます。ほとんどの場合、consumerはoffsetを線形に進めますが、任意のoffsetから開始して任意の順序でメッセージを読み取ることもできます。

* Consumerは通常、offsetをKafka Clusterに「commit」して、consumerが中断したところから再開できるようにします（例：再起動した場合）。

=== Consumerの実行
* *下部のターミナル* で、以下のコマンドを使用してOpenShift Clusterにログインします。image:common/clipboard.png[width=30px]アイコンを使ってコピーしてください。
+
[source,sh,role="execute",subs=attributes+]
----
{login_command}
----

* *下部のターミナル* で以下のコマンドを実行して、作成したtoolboxコンテナに接続します。
+
[source,sh,role="execute",subs=attributes+]
----
oc rsh -n kafka-{user_name} kafka-toolbox
----

* *下部のターミナル* でconsumerを実行します。
+
[source,sh,role="execute",subs=attributes+]
----
kafka-console-consumer.sh --bootstrap-server $BOOTSTRAP --topic demo \
  --group demo-group \
  --from-beginning \
  --property print.key=true \
  --property print.partition=true \
  --property key.separator=" | "
----



* producerが以前に送信したメッセージがconsumerターミナルに表示され、consumeされていることを確認します。 *null値* は、メッセージ送信時にkeyが提供されなかったことを示しています。


* producerターミナルでさらにメッセージを入力し、consumerターミナルでconsumeされることを確認します。

* keyを提供していないため、メッセージはpartition間に分散される *可能性があり* 、その場合は順序が保証されません。次のセクションでは、partitionとkeyを使用して順序を保証する方法を見ていきます。
+
NOTE: 送信するメッセージが非常に少ないため、すべてのメッセージが同じpartitionに入る可能性があります。しかし、実際の環境では1秒あたり数千のメッセージがあり、keyがない場合、メッセージは異なるpartitionに入る可能性があります。

* https://amq-streams-console-{user_name}.{openshift_cluster_subdomain}/kafka[streams console, window="_amqstreams"] > Consumer groupsに移動します。*demo-group* consumer groupをクリックして詳細を表示します。
+
image::m1/console_demo-group.png[]

* 各consumerはconsumer groupに属します。consumer groupはconsumerインスタンスのリストで、フォールトトレランス（耐障害性）とスケーラブルなメッセージ処理を保証します。consumer groupについては次のセクションで詳しく学習します。

* グループ内の唯一のconsumer（consumer ID: console-consumer）をクリックして詳細を表示します。consumer groupに1つのconsumerのみが含まれている場合、そのconsumerがすべてのpartitionのすべてのメッセージを処理する責任を持ちます。
+
image::m1/console_consumer.png[]

* メッセージがconsume済み（ *Committed* 列と *End Offset* 列が非ゼロ値）の任意のpartitionをクリックして詳細を表示します。
+
image::m1/console_consumed_partition.png[]

* そのpartitionからconsumerがconsumeしたメッセージを確認できるはずです。
+
image::m1/console_consumed_partition_messages.png[]
