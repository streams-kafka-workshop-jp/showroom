= showroom フォルダ構成ガイド

このドキュメントは、streams-kafka-workshop の Web ページコンテンツを含む `./showroom` フォルダの構造を説明します。

== 概要

`showroom` フォルダは Antora を使用した静的サイトジェネレータで、Kafka ワークショップのドキュメントサイトを構築します。

* *サイトタイトル*: The Kafka Workshop
* *ドキュメント形式*: AsciiDoc
* *ビルドシステム*: Antora
* *テーマ*: RHDP Showroom Theme

== ディレクトリ構造

----
./showroom/
├── .github/
│   └── workflows/
│       └── gh-pages.yml              # GitHub Pages へのデプロイワークフロー
├── content/                           # メインコンテンツディレクトリ
│   ├── antora.yml                     # Antora プロジェクト設定
│   ├── lib/                           # Antora 拡張機能
│   │   ├── all-attributes-console-extension.js
│   │   ├── attributes-page-extension.js
│   │   ├── dev-mode.js
│   │   └── unlisted-pages-extension.js
│   └── modules/
│       └── ROOT/                      # ルートモジュール
│           ├── nav.adoc               # ナビゲーション定義
│           ├── assets/
│           │   └── images/            # 画像リソース（カテゴリ別）
│           └── pages/                 # ドキュメントページ
├── utilities/                         # ビルド・開発用ユーティリティ
│   ├── lab-build                      # サイトビルドスクリプト
│   ├── lab-clean                      # クリーンアップスクリプト
│   ├── lab-serve                      # ローカルサーバー起動スクリプト
│   └── lab-stop                       # サーバー停止スクリプト
├── default-site.yml                   # Antora サイト設定
├── README.adoc                        # プロジェクト README
└── .gitignore
----

== 設定ファイル

=== antora.yml
プロジェクトのメタデータと属性を定義：

* プロジェクト名: modules
* タイトル: The Kafka Workshop
* バージョン: master
* 属性:
** `streams`: Red Hat Streams for Apache Kafka
** `registry`: Red Hat build of Apicurio Registry

=== default-site.yml
Antora サイトの全体設定：

* サイトタイトル: Streams Workshop
* UI バンドル: RHDP Showroom Theme (v0.0.1)
* 出力ディレクトリ: ./www
* コンテンツソース: ./content

== 開発ワークフロー

=== ユーティリティスクリプト

. *lab-build*: サイトをビルド
. *lab-serve*: ローカル開発サーバーを起動
. *lab-stop*: サーバーを停止
. *lab-clean*: ビルド成果物をクリーンアップ

=== GitHub Actions

`.github/workflows/gh-pages.yml` により、GitHub Pages への自動デプロイが設定されています。

== Antora 拡張機能（lib/）

* *all-attributes-console-extension.js*: コンソール属性の拡張
* *attributes-page-extension.js*: ページ属性の拡張
* *dev-mode.js*: 開発モード機能
* *unlisted-pages-extension.js*: 非表示ページの管理

== 対象トピック

このワークショップでは以下のトピックをカバー：

* Apache Kafka の基礎概念

* Debezium を使用した Change Data Capture
* Apicurio Schema Registry の活用
* Kafka のセキュリティ（認証・認可）
